# 儿童积分换购-页面设计规范（桌面优先）

## 全局设计（适用于所有页面）
### Theme
- 主题：疯狂动物城（Zootopia）
- 主角：兔子警官（Judy Hopps 风格形象）
- 视觉元素：城市天际线、警徽/警帽、对勾与星星成就、轻量漫画质感插画
 
### Layout
- 桌面端：最大内容宽度 1200px，居中；页面主体采用 `CSS Grid`（12 列）+ 局部 `Flex`。
- 断点：>=1024px（桌面）；<1024px（平板/手机）改为单列堆叠，列表卡片改为 2 列/1 列。

### Meta Information（默认）
- Title 模板：`{页面名} - 儿童积分换购`
- Description：`用积分兑换喜欢的礼物；家长密钥解锁加分。`
- Open Graph：`og:title` 与 Title 一致；`og:description` 与 Description 一致；`og:type=website`。

### Global Styles（Design Tokens）
- 背景色：`#F6F8FB`（偏明亮城市雾蓝）
- 内容卡片：`#FFFFFF`
- 主题色（按钮/高亮）：`#2F6BFF`（警灯蓝）
- 辅助强调色：`#F59E0B`（徽章金）
- 成功/警告/错误：`#16A34A / #F59E0B / #DC2626`
- 字体：系统字体；字号：14/16/20/28（辅助/正文/小标题/主标题）
- 圆角：卡片 12px；按钮 10px
- 按钮：主按钮实心主题色；hover 亮度 +6%；禁用态透明度 50%
- 链接：主题色下划线 hover

## 页面 1：积分换购首页（/）
### Meta
- Title：`积分换购 - 儿童积分换购`

### Page Structure
- 顶部导航（横向） + 主内容（两栏）
- 左栏：积分与加分入口
- 右栏：换购商品列表

### Sections & Components
1. 顶部导航（NavBar）
   - 左：产品名“积分换购”
   - 右：入口“商品管理”（仅管理员可见/可点击）；用户状态区（未登录显示“登录”按钮；已登录显示昵称/退出）
2. 积分概览卡（PointsCard）
   - 显示：`当前积分：{points_balance}`
   - 状态提示：最近一次“加分/换购”的结果（成功/失败文案）
3. 加分入口卡（AddPointsEntryCard）
   - 说明文案：`家长入口：需要密钥才能加分`
   - 主按钮：`进入加分`
   - 点击后跳转 `/points`，进入家长密钥校验
4. 换购列表（ProductGrid）
   - 桌面端：3 列卡片网格；卡片包含主图、名称、所需积分、进入详情按钮/整卡可点击
   - 空态：无商品时提示“暂无可换购商品”

### Interaction
- 进入加分：按钮 hover 有轻微上浮；进入 `/points` 后先进行家长密钥校验。

## 页面 1.5：加分页面（/points）
### Meta
- Title：`加分 - 儿童积分换购`

### Page Structure
- 顶部导航 + 主内容（单列）
- 首屏为“家长密钥校验”卡片；校验通过后展示“任务卡片区 + 自定义加分区”

### Sections & Components
1. 家长密钥卡（ParentUnlockCard）
   - 输入框：`家长密钥`（支持粘贴，默认密码态）
   - 主按钮：`解锁加分`
   - 失败提示：`密钥错误，请重试`
2. 任务加分卡片区（TaskPointsGrid）
   - 以 2-3 列网格展示可选卡片
   - 每张卡片包含：插画（疯狂动物城主题）、标题、`+X` 积分角标、轻提示文案（如“点击加分”）
   - 点击卡片后：弹出确认（可选）或直接加分并显示“成功动效”（星星/对勾/徽章）
3. 自定义加分（CustomPointsCard）
   - 输入：分值（必填，步进 +/-）
   - 输入：原因（可选）
   - 提交按钮：`确认加分`

### Task Cards（固定内容与插画要求）
- 数学题 `+3`：兔子警官在做算术/写板书
- 读中文绘本 `+1`：兔子警官在看带汉字封面的绘本
- 英文绘本 `+1`：兔子警官在看写有“ABC/English”的绘本
- 自觉洗漱 `+1`：兔子警官在洗脸刷牙，泡泡元素
- 穿衣服 `+1`：兔子警官在整理衣服/穿外套
- 老师表扬 `+5`：兔子警官拿奖状/被老师点赞，星星奖章元素

## 页面 2：商品详情页（/product/:id）
### Meta
- Title：`商品详情 - 儿童积分换购`
- Open Graph：`og:image` 优先使用商品主图

### Page Structure
- 顶部导航 + 主内容（左图右文，桌面端两列）

### Sections & Components
1. 面包屑（Breadcrumbs）
   - `积分换购首页 / 商品详情`
2. 图片区（Gallery）
   - 主图大图 + 缩略图条（多图时可切换）
3. 信息区（InfoPanel）
   - 商品名称、描述
   - 所需积分（强调样式）
   - 主按钮：“立即换购”
   - 辅助文案：显示“你的积分：X”，当 X < 所需积分时展示“积分不足”并禁用按钮
4. 换购确认弹窗（RedeemConfirmModal）
   - 文案：确认扣减积分
   - 操作：确认/取消；确认后显示结果（成功/失败）

## 页面 3：商品管理页（管理员）（/admin/products）
### Meta
- Title：`商品管理 - 儿童积分换购`

### Page Structure
- 顶部导航 + 管理面板（单列卡片堆叠）
- 重点突出“新增商品”和“图片管理（上传/推荐）”两个区块

### Sections & Components
1. 权限提示（GuardBanner）
   - 非管理员：显示“无权限访问”与返回首页按钮
2. 新增商品表单（CreateProductForm）
   - 字段：商品名称（必填）、所需积分（必填，数字）、描述（可选）
   - 提交按钮：“创建商品”
   - 成功提示：创建成功并进入“图片管理”区块
3. 图片管理（ProductImageManager）
   - 目标商品选择：下拉选择刚创建的商品（或默认选中最新创建）
   - 已有图片列表：网格预览；可设置主图（单选）
   - 图片上传（UploadPanel）：拖拽区/选择文件按钮；上传进度条
4. 自动推荐图片（RecommendPanel）
   - 触发按钮：“自动推荐 3 张图片”
   - 候选区：固定展示 3 张候选图卡片（大图预览 + 单选/多选勾选）
   - 保存按钮：“保存所选图片到商品”

### Interaction
- 推荐流程：点击触发后按钮 loading；返回 3 张候选图；允许不选直接关闭；保存成功后合并到“已有图片列表”。
- 上传流程：上传完成立即可预览；若设置为主图则更新封面。
